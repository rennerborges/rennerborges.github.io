Básico do escalonador

  • Prioridade;
  • Quantum;
  • Threads.

Curiosidades do Linux

  • Considera que os processos só existem enquanto grupos de threads;
  • Escalona apenas threads.

Metas do escalonador Linux

  • Escalonamento de tempo-real;
  • Interatividade;
  • Eficiência;
  • Justiça e prevenção de Starvation;

Requisitos do escalonamento do Linux

  • Apresentar boa performance em programas interativos, mesmo com carga elevada;
  • Distribuir de maneira justa o tempo da CPU;
  • O escalonador é preemptivo;
  • Cada processo possui uma prioridade, recalculada dinamicamente;

Requisitos do escalonamento do Linux

  • Ser eficiente em máquinas SMP (Symmetric Multi Processor);
  • Possuir suporte para tempo real;
  • O escalonador entrega a CPU para o processo que possui a maior prioridade.

Categorias de processos considerados

  • Processos interativos (time-sharing);
  • Processos em tempo real.

Processos de tempo real

  • FCFS;
  • Round Robin.

Kernel 1.0.x

  • Escalonamento baseado em threads;
  • Toda tarefa pega um número fixo de quantum;
  • O escalonador verifica todos os processos;
  • Linux era somente de console.

Kernel 2.4

  • Poucas diferenças quando comparado ao anterior;
  • Principais diferenças;
    • Classes de escalonamento
    • Conceito de “bondade”
    • A escolha do escalonador é feita com base na classe de escalonamento

Kernel 2.6

  • Complexidade O(1);
  • Tenta identificar os processos interativos;
  • Código do algoritmo 3x maior que o anterior;
  • Todas as tarefas em execução separadas por prioridade;
  • O algoritmo sempre escolhe o array de maior prioridade.

Kernel 2.6.23

  • Reescrita completa;
  • Algoritmo mais simples;
  • Não utiliza os conceitos de
    • Tempo de sleep
    • Identificação de processo interativo
    • Array de prioridade
  • Utiliza árvore rubro-negra;
